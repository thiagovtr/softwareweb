# Product Backlog

## 1. Visão geral
O Product Backlog reúne as funcionalidades, necessidades e melhorias da solução proposta.

## 2. Estratégia de priorização
A priorização foi definida com base no fluxo essencial para que o aplicativo funcione. A prioridade máxima (Alta) foi dada à segurança do acesso (cadastro apenas com e-mail institucional) e às ações principais que resolvem o problema: oferecer e buscar caronas. Funcionalidades complementares, como a reserva efetiva da vaga e o sistema de avaliações, receberam prioridade média e baixa, sendo alocadas para as sprints seguintes.

## 4. Exemplo de história de usuário
**US01**  
Como **recepcionista**, quero **registrar um novo atendimento**, para **organizar a fila de encaminhamento**.

**Critérios de aceitação:**
- deve ser possível cadastrar nome e dados principais;
- o sistema deve registrar data e hora;
- o atendimento deve aparecer em uma lista de acompanhamento.
